# リポジトリーの説明
- 本リポジトリーはKotlin, Spring Bootで実装するウェブアプリケーションサンプルです。
- Gitのリポジトリールートはこのファイルがあるディレクトリーより上位にありますが、本プロジェクトのルートディレクトリーはこのファイルが置かれているディレクトリーです。ここをプロジェクトルートディレクトリーと呼びます。明示的に指示した場合を除いてプロジェクトルートディレクトリー配下のファイルのみを対象に作業してください
  - 指示する際にはプロジェクトルートを `/` とする場合があります。例えば本ファイルは `/AGENTS.md` と呼称する場合があります。

# 本リポジトリーで管理するプロジェクトの目的

現実に発生している/発生し得る、タイムゾーンの問題を理解し、解決策を決定するためにコードを実装しながら検討していくのが目的です。

具体的な検討事項は、作業を進める過程で変わっていくので都度説明していきます。

# アーキテクチャ

- DDDのレイヤーアーキテクチャーに従って実装してください
  - 作成・編集したファイルが間違っている場合は都度指示します。ですので、基本的なディレクトリーやファイル構成は既存のものを参考にできるはずです
    - ただし、ミスで既存の構造も間違っている場合があります。その場合は指摘してください。
    - ルールが間違っている場合は、AGENTS.mdを更新していく予定です

# コーディング作業に当たって守るべき事項

- ファイルの編集作業が発生する場合、 `/docs/plan.md` の手順に従って進めてください。
